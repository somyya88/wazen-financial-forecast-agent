# Wazen CFO Intelligence Agent — V12.6 File-aware Analysis Core

## لماذا هذا التحديث؟
كان النموذج يتعامل مع ملفات المبيعات والمصروفات كأنها شرط لبناء التحليل، مع أن ميزان المراجعة وحده يكفي لاستخراج قائمة دخل ومركز مالي ونسب مالية أساسية. كذلك عند رفع ملفات سنة حالية وسنة سابقة، كان يمكن اختيار ملف سابق كملف نشط بالخطأ.

## التغييرات الأساسية

### 1. ميزان المراجعة أصبح كافيًا لبناء Level A
- Trial Balance وحده يبني:
  - قائمة الدخل.
  - المركز المالي التحليلي.
  - نسب الربحية والسيولة والمديونية الأساسية.
  - التحليل الرأسي والأفقي حيثما أمكن.
- ملفات المبيعات والمصروفات والسيولة أصبحت ترفع عمق التحليل ولا توقفه عند غيابها.

### 2. اختيار الملف النشط أصبح حسب أحدث سنة/فترة
- عند رفع ميزان 2025 وميزان 2026، يستخدم النموذج 2026 كالفترة الحالية.
- يحتفظ بملفات السنة السابقة للمقارنة المستقبلية، ولا يجعلها مصدر الفترة الحالية بالخطأ.

### 3. محرك ميزان المراجعة أصبح أكثر مرونة
- قراءة الإيرادات لم تعد تعتمد فقط على اسم حساب محدد أو كود حساب محدد.
- أضيفت fallback rules لقراءة:
  - صافي المبيعات.
  - الإيرادات الأخرى.
  - تكلفة المبيعات / تكلفة الإيراد.
  - المصروفات التشغيلية.
- عند غياب أعمدة الحركة، يستخدم أرصدة آخر المدة كبديل بدلاً من إخراج أرقام صفرية.

### 4. P&L Engine أصبح File-aware
- يستخدم ميزان المراجعة كمصدر أساسي عند توفره.
- إذا فشل الميزان في قراءة الإيرادات لكن ملف المبيعات متوفر، يبني P&L هجين بدل عرض صافي إيرادات = 0.
- إذا غاب ملف المصروفات، يستخدم مصروفات ميزان المراجعة.

### 5. جاهزية التحليل أصبحت تشرح نطاق التحليل
- تعرض ما يمكن استخراجه الآن من الملفات الحالية.
- تفرق بين:
  - Financial Analysis من الميزان.
  - CFO Analysis عند توفر السيولة والذمم.
  - Strategic CFO Analysis عند توفر المبيعات التفصيلية والعملاء والأصناف.

### 6. مسار البناء لم يعد يوقف التحليل الكامل بسبب نقص ملف تفصيلي
- الحد الأدنى الآن: ميزان مراجعة، أو مبيعات + مصروفات كبداية تشغيلية.
- التحليل لا يتوقف بغياب السيولة أو أعمار العملاء؛ بل يوضح ما لا يمكن الحكم عليه بثقة.
